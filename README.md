ES UN ARCHIVO PHYTON CREADO PARA REVISAR A TRAVES DE LA LIBRERIA SOCKETS LOS PUERTOS DE UNA DIRECCION IP.

SE DEBE DE PONER UNA DIRECCION IP DE LA QUE SEAMOS DUEÑOS EN LA VARIABLE ip_objetivo
SE DEBE DE DAR UN RANGO DE PUERTOS A EVALUAR, EN EL CASO DE ESTE SCRIPT ES DEL 1 AL 200.

A CONTINUACION PARA REVISAR SI LOS PUERTOS ESTAN ABIERTOS SE ABRE UN CICLO FOR, EN EL CUAL SE VAN REVISANDO 1 A UNO CADA PUERTO
SI EL PUERTO ESTA CERRADO EL CICLO CONTINUA CON EL SIGUIENTE PUERTO A REVISAR Y ASI SUCESIVAMENTE.

CUANDO UN PUERTO SE ENCUENTRA ABIERTO, ESTE SE DETERMINA POR MEDIO DE LA LIBRERIA socket con el comando connect_ex()
SI EL PUERTO SE ENCUENTRA ABIERTO ENTRAMOS A UN CICLO TRY, 
EN EL CUAL SE REVISA LA CONEXION DE CADA PUERTO CON OTRO COMANDO DE LA LIBRERIA SOCKET,
socket_conexion.settimeout(), INDICAMOS LA CANTIDAD DE SEGUNDOS QUE DEBE DEMORAR EL ANALISIS DEL PUERTO PARA PODER LEERLO 
conexion-recv(), EN EL CUAL DEBEMOS DE INDICAR LA CANTIDAD DE BYTES.

UNA VEZ ADQUIRIDA ESTA INFORMACION SE ALMACENA CON LA VARIABLE BANNER Y ASI MISMO SE DEBE DECODIFICAR.

NO SIMPRE SERA POSIBLE LEER LA INFORMACION, PUES HABRA VECES QUE LA INFORMACION PARA DECOFICIAR TENDRA QUE SER CON OTRA
FUNCION DE decode QUE NO SEA 'utf-8', O TENDREMOS QUE AUMENTAR LA CANTIDAD DE TIEMPO DE DEMORA DE ANALISIS, O AMPLIAR EL RANGO DE PUERTOS

FINALMENTE SI EL PUERTO ANALIZADO EN EL CICLO TRY NO SE PUEDE LEER SE IMPRIMIRA COMO BANNER: NO DISPONIBLE, PERO SI ES POSIBLE LEERLO
NOS ARROJARA LA LECTURA DEL BANNER.

Y UNA VEZ QUE SE ANALISE TODO EL RANGO Y LOS PUERTOS ABIERTOS SE IMPRIMIRA EL FINAL DE ESCANEO COMPLETADO.

ES IMPORTANTE INTENTAR CON VARIAS DIRECCIONES IP DIFERENTES (DE LAS QUE TENGAMOS AUTORIA PARA NO HACER ALGO ILEGAL)
PORQUE NO SIEMPRE SE PUEDEN DETERMINAR TODOS LOS PUERTOS EN UNA IP, HAY QUE INTENTAR EN VARIAS.


FINALMENTE ES IMPORTANTE RECAPITULAR QUE CON ESTE CODIGO PY, SE PUEDE APRENDER A UTILIZAR ALGUNOS COMANDOS DE ANALISIS Y
A ARMAR CODIGO PYTHON PARA ANALIZAR PURTOS Y DIRECCIONES IP.
